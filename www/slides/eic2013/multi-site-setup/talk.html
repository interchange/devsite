What is Multi-store?

  • One Interchange daemon, many sites
  • Shared images, configuation between sites
  • Different pricing or logic between sites
  • Different look and feel per site

There are 4 different areas that we setup to make this happen:

  • Apache
  • Interchange catalog
  • Interchange global
  • database 
Standard directory tree

user
|-- live
|   |-- bin 
|   |-- catalogs
|   |   |-- site1
|   |   |-- site2
|   |   |-- site3
|   |   `-- shared
|   |-- cgi-bin
|   |-- htdocs
|   |   |-- master_images
|   |   |-- site1
|   |   |-- site2
|   |   |-- site3
|   |   |-- shared_html
|   |-- interchange

Catalogs directory tree looks something like this:

|-- site1 
|   |-- captcha -> ../../htdocs/site1/images/captcha/
|   |-- include
|   |   |-- user_mgmt -> ../../shared/include/user_mgmt
|   |-- pages
|   |   |-- flypage.html -> ../../shared/pages/flypage.html
|   |-- products
|   |   |-- shared_variable.txt -> ../../shared/products/shared_variable.txt
`-- shared
    |-- include
    |   |-- user_mgmt
    |-- pages
    |   |-- flypage.html
    |-- products
    |   |-- shared_variable.txt

Add to catalog.cfg:

VariableDatabase shared_variable

Interchange Global adjustmenets

  • Take advantage of Interchange global space
  • Shared usertags, actions maps and configuration variables

Database Setup Strategies

  • No 'right' way to do it
  • Different users per catalog
  • Shared database tables
  • Custom views
