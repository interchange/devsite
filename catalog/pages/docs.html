Building documentation from source

You can browse the documentation source repository using our GitHub repository. To build the documentation yourself, you need to clone your own local copy, as follows: 

git clone https://github.com/interchange/xmldocs.git

You should then read the README file for build instructions and a list of prerequisites.

Helping with the documentation

There are many ways how you can help with the documentation.

You can fork our xmldocs project on GitHub and publish changes in your own fork. For an introduction to our build system and ways to write documentation, see the README and WRITING files. To get more deeply involved in __DOCBOOK__ (DocBook XML in particular), see __DOCBOOK__ and Sagehill.net.
