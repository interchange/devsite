Interchange Documentation Set

The traditional documentation set is available on http://www.icdevgroup.org/interchange-doc-5.2.0/.

You might be aware that the documentation can also be found in the Interchange 'docs' CVS module, but should only be used in the transition period until the new documentation is officially released.

XMLDOCS - New Interchange Documentation

In 2004., __docelic__ of __ICDEVGROUP__ started a complete __IC__ documentation redesign; the effort is known as the xmldocs project.

xmldocs set is based on __DOCBOOK__ XML. Large ammount of documentation is generated by special scripts that parse __IC__ source and extract key information for any particular __IC__ release. xmldocs were also designed with "profiling" in mind, a convenient feature allowing us to produce documentation sets "optimized" for each target distribution (tarball, Debian GNU, Red Hat ...). xmldocs also support on-line examples when you read the documentation directly off of our web site.

The ALPHA XMLDOCS documentation is available on-line at /xmldocs/.

You can browse __XMLDOCS_CVSWEB__ documentation sources on-line using __CVSWEB__. To build it yourself, you need to check-out a copy from the CVS to your local machine: 

cvs -z6 -d :pserver:cvs@cvs.icdevgroup.org:/var/cvs \
  checkout -P xmldocs
Then read file README for build instructions and pre-requirements.

There are many ways how you can help with the documentation.

You can check-out a copy from CVS as shown above and work on providing us with diff patches to the CVS. For an introduction to our build system and ways to write documentation, see files README and WRITING files. To get more deeply involved in __DOCBOOK__ (DocBook XML in particular), see __DOCBOOK__ and Sagehill.net.
