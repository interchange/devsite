DOCS - Old Interchange documentation

The traditional (old) documentation set is available on http://www.icdevgroup.org/interchange-doc-5.2.0/.

You might be aware that the documentation sources can also be found in the Interchange 'docs' CVS module, but should only be used in the transition period until the new documentation is officially released.

XMLDOCS - New Interchange documentation

In 2004 and 2005, __ICDEVGROUP__ started a complete __IC__ documentation redesign; the effort is known as the XMLDOCS project.

xmldocs set is based on __DOCBOOK__ XML. Large amount of documentation is generated by special scripts that parse __IC__ source and extract key information for any particular __IC__ release. xmldocs were also designed with "profiling" in mind, a convenient feature allowing us to produce documentation sets "optimized" for each target distribution (tarball, Debian GNU, Red Hat ...). xmldocs also support on-line examples when you read the documentation directly off of our web site.

Finally, note that our on-line documentation pages also contain feedback links; do not miss the great opportunity to open your account, submit comments and information requests, and rate existing documentation.

Reading xmldocs documentation on-line

The ALPHA-quality XMLDOCS documentation is available on-line in the following flavors:

  • Generic documentation, static
  • Generic documentation, with on-line examples

Downloading xmldocs documentation

The documentation is available for download in the following flavors:

  • Generic (tarball) documentation, static
  • Debian GNU-profiled documentation, static
  • Red Hat-profiled documentation, static

Building documentation from source

You can browse __XMLDOCS_CVSWEB__ documentation sources on-line using __CVSWEB__. To build it yourself, you need to check-out a copy from the CVS to your local machine: 

cvs -z6 -d :pserver:cvs@cvs.icdevgroup.org:/var/cvs \
  checkout -P xmldocs
Then read file README for build instructions and pre-requirements.

Helping with the documentation

There are many ways how you can help with the documentation. [comment] The simplest one includes browsing our on-line documentation and submitting feedback forms found at the bottom of the pages. Using feedback forms, you can notify us of smaller errors, request clarification, or you can express your satisfaction with the page content.

As you might know, we're currently in the process of wading through the interchange-users mailing list archives in search of useful information that never made it to the documentation. This requires an enormous amount of work, due to large number of user posts and spam that added up over the years. We want to bring attention of the users' community to the problem. By registering on our site, you obtain access to a conveniently sorted lists of mailing list posts which allow you to quickly read and rate individual posts. By doing a good work (eliminating spam and "dry" messages, or bringing our attention to useful posts) you will not only help us a great deal, but also earn yourself a recognition and public acknowledgement with __ICDEVGROUP__. [/comment]

You can check-out a copy from CVS as shown above and work on providing us with diff patches to the CVS. For an introduction to our build system and ways to write documentation, see files README and WRITING. To get more deeply involved in __DOCBOOK__ (DocBook XML in particular), see __DOCBOOK__ and Sagehill.net.
